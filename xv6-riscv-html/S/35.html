  31 // per-process data for the trap handling code in trampoline.S.
  32 // sits in a page by itself just under the trampoline page in the
  33 // user page table. not specially mapped in the kernel page table.
  34 // the sscratch register points here.
  35 // uservec in trampoline.S saves user registers in the trapframe,
  36 // then initializes registers from the trapframe's
  37 // kernel_sp, kernel_hartid, kernel_satp, and jumps to kernel_trap.
  38 // usertrapret() and userret in trampoline.S set up
  39 // the trapframe's kernel_*, restore user registers from the
  40 // trapframe, switch to the user page table, and enter user space.
  41 // the trapframe includes callee-saved user registers like s0-s11 because the
  42 // return-to-user path via usertrapret() doesn't return through
  43 // the entire kernel call stack.
